The Lux Explorer runs on Kubernetes with Docker containers. Five explorer instances serve different chains from the same codebase, each with its own PostgreSQL database and chain-specific branding.

## Architecture

```
Traefik Ingress
  explore.lux.network       -> blockscout-mainnet:4000
  explore-zoo.lux.network   -> blockscout-zoo:4000
  explore-hanzo.lux.network -> blockscout-hanzo:4000
  explore-spc.lux.network   -> blockscout-spc:4000
  explore-pars.lux.network  -> blockscout-pars:4000
```

## Explorer Instances

| Instance | Domain | Chain | Backend RPC |
|----------|--------|-------|-------------|
| C-Chain | `explore.lux.network` | Mainnet C-Chain | `luxd-1:9630/ext/bc/C/rpc` |
| Zoo | `explore-zoo.lux.network` | Zoo subnet | `luxd-1:9630/ext/bc/{id}/rpc` |
| Hanzo | `explore-hanzo.lux.network` | Hanzo subnet | `luxd-1:9630/ext/bc/{id}/rpc` |
| SPC | `explore-spc.lux.network` | SPC subnet | `luxd-1:9630/ext/bc/{id}/rpc` |
| Pars | `explore-pars.lux.network` | Pars subnet | `luxd-1:9630/ext/bc/{id}/rpc` |

## Memory Optimization

For small chains, set these environment variables to prevent OOM crashes:

```
INDEXER_DISABLE_COIN_BALANCE_FETCHER=true
INDEXER_DISABLE_BLOCK_REWARD_FETCHER=true
INDEXER_DISABLE_EMPTY_BLOCKS_SANITIZER=true
POOL_SIZE=3
POOL_SIZE_API=2
INDEXER_MEMORY_LIMIT=1gb
```

## Kubernetes Requirements

- Container memory limit: 8Gi per instance
- Deployment strategy: `Recreate` (not `RollingUpdate`)
- PostgreSQL with chain-specific `chain` label on Service selector
- TLS via Traefik certResolver with letsencrypt
