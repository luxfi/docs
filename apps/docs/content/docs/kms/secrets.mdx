Lux KMS provides centralized secret management with automatic versioning, rotation, and audit logging. Secrets are encrypted at rest with organization-level Key Encryption Keys (KEKs).

## Features

- **Versioning** -- Every secret change creates a new version with rollback support
- **Rotation** -- Automatic rotation with configurable intervals per secret
- **Folders** -- Organize secrets in a hierarchical folder structure
- **Tags** -- Label and filter secrets for access control
- **Audit** -- Full audit log of all secret access and modifications
- **Point-in-Time Recovery** -- Restore entire project state to any previous point

## Create a Secret

```bash
curl -X POST https://kms.lux.network/api/v1/secrets \
  -H "Authorization: Bearer $KMS_TOKEN" \
  -d '{
    "secretName": "db-password",
    "secretValue": "s3cureP@ss",
    "environment": "production",
    "type": "shared"
  }'
```

## Environment Scoping

Secrets are scoped per environment. Each environment maintains its own values:

| Environment | Description |
|-------------|-------------|
| `development` | Local development values |
| `staging` | Pre-production testing |
| `production` | Live production secrets |

## Secret Rotation

Configure automatic rotation for database credentials:

```json
{
  "secretName": "postgres-password",
  "rotationInterval": "30d",
  "rotationProvider": "postgresql",
  "rotationConfig": {
    "host": "db.internal",
    "port": 5432,
    "database": "myapp"
  }
}
```

## Kubernetes Integration

The KMS Kubernetes operator syncs secrets to K8s:

```yaml
apiVersion: secrets.lux.network/v1alpha1
kind: KMSSecret
metadata:
  name: app-secrets
spec:
  secretsPath: /myproject/production
  managedSecretReference:
    secretName: app-secrets
    secretNamespace: default
```
