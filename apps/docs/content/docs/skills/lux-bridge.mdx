## Overview

Lux Bridge is a **decentralized cross-chain bridge** using Multi-Party Computation (MPC) for secure asset transfers between Lux subnets and external EVM chains. It is a pnpm monorepo with a Next.js/React frontend, Express API server, and shared TypeScript packages.

## When to use

- Building or deploying cross-chain asset transfer infrastructure on Lux
- Integrating MPC-based threshold signing into bridge workflows
- Running a bridge UI, explorer, or API server for Lux Network

## One-file quickstart

```bash
git clone https://github.com/luxfi/bridge.git
cd bridge
pnpm install

# Start infrastructure (Lux ID, KMS, NATS, Consul, PostgreSQL)
make up

# Start MPC network (3-node, 2-of-3 threshold)
make start-mpc-nodes

# Start API server
cd app/server && pnpm dev

# Start bridge UI (separate terminal)
cd app/bridge && pnpm dev
```

## Architecture

```
Bridge UI (port 3000) --> Bridge Server (port 5000) --> MPC Nodes --> Blockchain Networks
                                                           |
                                              KMS + Lux ID + Consul
```

### Infrastructure services (via compose.yml)

| Service | Port | Purpose |
|---------|------|---------|
| Lux ID (Casdoor) | 8000 | Unified authentication |
| KMS (Vault) | 8200 | Key management |
| NATS | 4223 | Messaging |
| Consul | 8501 | Service discovery |
| PostgreSQL | 5433 | Bridge database |
| Lux ID DB | 5434 | Auth database |

### MPC network

| Node | HTTP | gRPC |
|------|------|------|
| Node 0 | 6000 | 9090 |
| Node 1 | 6001 | 9091 |
| Node 2 | 6002 | 9092 |

## Key features

- **MPC Security**: 2-of-3 threshold signatures for bridge transactions
- **Multi-Chain**: Support for EVM and non-EVM chains
- **Unified Auth**: Lux ID (Casdoor) integration
- **Key Management**: Secure key storage via KMS (Vault)
- **Service Discovery**: Consul for dynamic configuration
- **Transaction Explorer**: Built-in explorer for tracking bridge operations

## Development

```bash
# Install dependencies
pnpm install

# Run tests
pnpm test

# Run tests across all packages
pnpm -r test

# Stop infrastructure
make down

# Stop MPC nodes
make stop-mpc-nodes
```
