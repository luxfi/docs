## Overview

Lux HSM (`github.com/luxfi/hsm`) is a **unified Go library** for hardware security module, KMS, and custody operations across the Lux ecosystem. It provides a single entry point for password providers (symmetric KMS decrypt), signing providers (asymmetric sign/verify), post-quantum ML-DSA signatures, threshold key share storage with AES-256-GCM encryption, and HSM-attested threshold signing.

## Hard requirements

1. ALWAYS use `github.com/luxfi/*` packages -- NEVER `go-ethereum` or `ava-labs`
2. NEVER store key shares in plaintext -- always use `KeyShareVault` with AES-256-GCM
3. NEVER hardcode passwords -- use PasswordProvider interface (cloud KMS or env/file)

## Supported Providers

### Signer Providers (Asymmetric Sign/Verify)

| Provider | Type | Algorithm | File |
|----------|------|-----------|------|
| `aws` | AWS KMS | ECDSA_SHA_256 | `signer_aws.go` |
| `gcp` | Google Cloud KMS | EC_SIGN_P256_SHA256 | `signer_gcp.go` |
| `azure` | Azure Key Vault | ECDSA P-256 | `signer_azure.go` |
| `zymbit` | Zymbit SCM | ECDSA P-256 (PKCS#11) | `signer_zymbit.go` |
| `mldsa` | Post-Quantum | ML-DSA-44/65/87 (FIPS 204) | `signer_mldsa.go` |
| `local` | In-Memory | ECDSA P-256 (dev only) | `signer_local.go` |

### Password Providers (Symmetric Decrypt)

| Provider | Type | Purpose | Env Var |
|----------|------|---------|---------|
| `aws` | AWS KMS Decrypt | ZapDB password decryption | `MPC_HSM_KEY_ID`, `AWS_REGION` |
| `gcp` | GCP KMS Decrypt | ZapDB password decryption | `GCP_PROJECT_ID`, `GCP_KMS_*` |
| `azure` | Azure Key Vault | Key unwrap | `AZURE_VAULT_URL`, `AZURE_KEY_*` |
| `env` | Environment Variable | Direct password read | `LUX_MPC_PASSWORD` |

## Core Interfaces

### Signer

```go
type Signer interface {
    Sign(ctx context.Context, keyID string, message []byte) ([]byte, error)
    Verify(ctx context.Context, keyID string, message, signature []byte) (bool, error)
    Provider() string
}
```

### PasswordProvider

```go
type PasswordProvider interface {
    GetPassword(ctx context.Context, keyID string) (string, error)
}
```

## One-file quickstart

### Installation

```go
import "github.com/luxfi/hsm"
```

### Basic Manager usage

```go
cfg := hsm.Config{
    PasswordProvider: "aws",   // or "gcp", "azure", "env", "file"
    PasswordKeyID:    "arn:aws:kms:us-east-1:...",
    SignerProvider:   "aws",   // or "gcp", "zymbit", "mldsa", "local"
    SignerKeyID:      "arn:aws:kms:us-east-1:...",
    Region:           "us-east-1",
}
mgr, err := hsm.New(cfg)

// Get ZapDB password from KMS
password, err := mgr.GetPassword(ctx)

// Sign a message
sig, err := mgr.Sign(ctx, message)

// Verify
ok, err := mgr.Verify(ctx, message, sig)
```

### Direct factory usage

```go
// Create signer directly
signer, err := hsm.NewSigner("mldsa", nil)
sig, err := signer.Sign(ctx, "my-key", message)

// Create password provider directly
pw, err := hsm.NewPasswordProvider("env", map[string]string{
    "env_var": "MY_DB_PASSWORD",
})
password, err := pw.GetPassword(ctx, "")
```

### Post-quantum signing (ML-DSA)

```go
signer := hsm.NewMLDSASigner(mldsa.MLDSA65) // NIST Level 3, 192-bit PQ security
sig, err := signer.Sign(ctx, "validator-0", message)
ok, err := signer.Verify(ctx, "validator-0", message, sig)
```

ML-DSA modes: `MLDSA44` (128-bit), `MLDSA65` (192-bit, recommended), `MLDSA87` (256-bit).

## Threshold Integration

### KeyShareVault

AES-256-GCM encrypted storage for threshold key shares. Encryption key derived from PasswordProvider:

```go
vault := hsm.NewKeyShareVault(passwordProvider, "kms-key-id")

// Store encrypted key share
vault.Store(ctx, "validator-0", share.Bytes(), meta)

// Load and decrypt
raw, meta, err := vault.Load(ctx, "validator-0")

// List, delete, get metadata without decryption
ids := vault.List()
vault.Delete("validator-0")
meta, err := vault.GetMeta("validator-0")
```

### HSMAttestingSigner

Wraps any `threshold.Signer` with HSM co-signatures. Every signature share is attested by the node's HSM key, binding shares to specific hardware:

```go
inner, _ := scheme.NewSigner(keyShare)
signer := hsm.NewAttestingSigner(inner, hsmSigner, "attestation-key")
share, _ := signer.SignShare(ctx, message, signers, nonce)

// Verify attestation
ok, _ := hsm.VerifyAttestation(ctx, hsmSigner, "attestation-key", share)
```

Wire format: `[4-byte share length][share bytes][attestation bytes]`

### ThresholdManager

Complete HSM-backed threshold signing solution combining vault + attestation:

```go
mgr, _ := hsm.NewThresholdManager(hsm.ThresholdConfig{
    PasswordProvider: "aws",
    PasswordKeyID:    "arn:aws:kms:...",
    SignerProvider:   "local",
    AttestKeyID:      "node-attest",
})

// Store key share (encrypted at rest)
mgr.StoreKeyShare(ctx, "validator-0", keyShare)

// Create attesting signer
signer, _ := mgr.NewSigner(ctx, "validator-0")
attestedShare, _ := signer.SignShare(ctx, msg, signers, nil)
```

### ThresholdHSM Interface

For HSMs with native threshold support (Zymbit SCM, custom FPGA):

```go
type ThresholdHSM interface {
    Signer
    NonceGen(ctx context.Context, keyID, sessionID string) (commitment []byte, nonceRef string, err error)
    SignThresholdShare(ctx context.Context, keyID, sessionID, nonceRef string, message []byte, signerIndices []int) ([]byte, error)
    ImportKeyShare(ctx context.Context, keyID string, share []byte) error
    ExportPublicShare(ctx context.Context, keyID string) ([]byte, error)
}
```

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| AWS KMS 400 error | Wrong key type | Use SIGN_VERIFY key (not ENCRYPT_DECRYPT) |
| Zymbit timeout | Device not connected | Check USB/I2C connection, port 6789 |
| ML-DSA key not found | Key auto-generated per ID | Keys live in memory, persist externally |
| Vault decryption fail | Password changed | Re-encrypt vault with new password |
| Attestation invalid | Wrong HSM key | Verify attestation key ID matches |
