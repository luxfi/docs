## Overview

Lux Quasar is the **post-quantum consensus engine** with Photonic Selection — a novel consensus protocol achieving sub-second 2-round finality with dual-certificate security (BLS aggregate + lattice post-quantum). This is a SEPARATE repository from the node.

**This is NOT Snow, Snowball, or Avalanche BFT.** Quasar is an original consensus protocol.

## Architecture

| Protocol | Role |
|----------|------|

| **QZMQ** | Post-quantum transport |

### Finality Flow

```
1. Transaction proposed
2. Photon selects committee (K-of-N)
3. Wave: threshold voting round
4. Round 1: BLS aggregate signature (fast finality)
5. Round 2: Lattice PQ certificate (quantum-safe finality)
6. Flare generates dual certificate
7. Transaction finalized (<1s)
```

### Dual Certificate Security

Every block requires TWO certificates to validate:
- **BLS12-381 aggregate signature** — fast classical security
- **Lattice post-quantum certificate** — quantum-safe assurance

Both must pass validation. This ensures security even if one cryptographic scheme is compromised.

### Chain Integration

All chains use Quasar:

- **C-Chain** — EVM-compatible contracts
- **X-Chain** — exchange/UTXO transactions
- **P-Chain** — platform/validator management
- **M-Chain** — MPC custody operations

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| Slow finality | Low peer count | Ensure sufficient connected peers |
| Consensus stall | Network partition | Check P2P connectivity |
| Certificate validation fail | Key mismatch | Verify BLS and lattice keys registered |
| Phase II timeout | Lattice computation slow | Check hardware requirements |
