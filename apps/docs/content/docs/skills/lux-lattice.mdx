## Overview

Lattice-based **multiparty homomorphic encryption** library. Full-RNS RLWE-based HE supporting BFV, BGV, and CKKS schemes with N-out-of-N and t-out-of-N threshold access structures. Pure Go, C++-competitive performance, WASM-compatible. Originally from EPFL Laboratory for Data Security, maintained by Lux Partners Limited.

## Encryption Schemes

### BFV (Fan-Vercauteren)

- **Type**: Exact integer arithmetic (scale-invariant)
- **Use case**: Counting, comparison, small integer circuits
- **Implementation**: Unified with BGV in `schemes/bgv/` package. Instantiate BGV evaluator with `ScaleInvariant=true`
- **Difference from textbook BFV**: Only `Mul`, `MulNew`, `MulRelin`, `MulRelinNew`, `Rescale` are replaced with scale-invariant variants
- **Constraint**: Plaintext modulus T must be coprime with ciphertext modulus Q (T cannot divide Q, otherwise not IND-CPA secure)
- **Noise**: ~0.5 bit less than textbook BFV due to rounded (not floored) RNS division

### BGV (Brakerski-Gentry-Vaikuntanathan)

- **Type**: Exact modular arithmetic
- **Use case**: Modular arithmetic circuits, integer SIMD
- **Package**: `github.com/luxfi/lattice/v7/schemes/bgv`
- **Plaintext space**: Z_T where T is the plaintext modulus
- **Slots**: For slot encoding, slots = largest n where T = 1 mod 2n. For coefficient encoding, slots = N
- **SIMD layout**: 2 x (N/2) matrix (Standard ring) or 1 x N matrix (Conjugate Invariant)
- **Operations**: Add, Sub, Mul, MulRelin, MulThenAdd, Rescale, column rotation, row rotation, inner sum, replicate, trace

**Example parameter set** (`ExampleParameters128BitLogN14LogQP438`):
```
LogN=14, Q=40+11*29 bits, P=40+39 bits, T=0x10001 (16-bit), 128-bit security
```

### CKKS (Cheon-Kim-Kim-Song)

- **Type**: Approximate real/complex arithmetic
- **Use case**: Floating-point ML inference, signal processing, statistics
- **Package**: `github.com/luxfi/lattice/v7/schemes/ckks`
- **Precision modes**: PREC64 (LogDefaultScale <= 64) and PREC128 (LogDefaultScale <= 128)
- **Levels consumed per rescale**: 1 (PREC64) or 2 (PREC128)
- **Slots**: N/2 complex (Standard ring) or N real (Conjugate Invariant)
- **Ring types**: Standard Z[X]/(X^N+1) or Conjugate Invariant Z[X+X^-1]/(X^2N+1) with bridge between them
- **Operations**: Add, Sub, Mul, MulRelin, Rescale, rotation, complex conjugation, inner sum, replicate, trace
- **Bootstrapping**: Full approximate CKKS bootstrapping in `circuits/ckks/bootstrapping/`

**Example parameter set** (`ExampleParameters128BitLogN14LogQP438`):
```
LogN=14, Q[0]=55-bit + 6*45-bit, P=2*55-bit, LogDefaultScale=45, 128-bit security
```

### Scheme-Agnostic Interface

The `schemes` package defines `Encoder` and `Evaluator` interfaces that all three schemes implement:

```go
type Evaluator interface {
    Add, AddNew, Sub, SubNew, Mul, MulNew, MulRelin, MulRelinNew,
    MulThenAdd, Relinearize, Rescale
}
```

## Package Architecture

### Ring Layer (`ring/`)

RNS-accelerated polynomial ring arithmetic:

| Component | Purpose |
|-----------|---------|
| `ring.Ring` | RNS polynomial ring with SubRing decomposition |
| `ring.Poly` | Polynomial with multi-level RNS coefficients |
| `ring.SubRing` | Single-prime modular arithmetic with NTT |
| `ring/ringqp/` | Q*P extended ring for key-switching |
| NTT | Number Theoretic Transform (Standard + Conjugate Invariant) |
| Samplers | Uniform, Ternary (fixed-H or probability), Discrete Gaussian |
| Basis Extension | RNS basis extension for cross-modulus operations |
| Modular Reduction | Montgomery and Barrett reduction |

**Ring types**: `Standard` = Z[X]/(X^N+1), `ConjugateInvariant` = Z[X+X^-1]/(X^2N+1)

### Multiparty Protocols (`multiparty/`)

RLWE-based multiparty homomorphic encryption with two access structures:

#### N-out-of-N Threshold

Secret key s = sum(s_i) where each party holds s_i. All N parties must collaborate. No trusted dealer needed.

| Protocol | Rounds | Purpose |
|----------|--------|---------|
| `PublicKeyGenProtocol` | 1 | Collective public key generation |
| `RelinearizationKeyGenProtocol` | 2 | Collective relinearization key (only 2-round protocol) |
| `GaloisKeyGenProtocol` | 1 | Collective Galois/rotation key |
| `EvaluationKeyGenProtocol` | 1 | Generic collective evaluation key |
| `KeySwitchProtocol` | 1 | Re-encrypt between known collective keys (noise flooding) |
| `PublicKeySwitchProtocol` | 1 | Re-encrypt to arbitrary public key (external receiver) |

#### t-out-of-N Threshold

Extension of N-out-of-N using Shamir Secret Sharing. Based on Mouchet et al. 2022 (ePrint 2022/780).

| Type | Purpose |
|------|---------|
| `Thresholdizer` | Generates Shamir polynomials and distributes shares |
| `ShamirPolynomial` | Degree-(t-1) polynomial with ringqp.Poly coefficients |
| `ShamirSecretShare` | A party's share S(alpha_i) |
| `ShamirPublicPoint` | Party identity (uint64) |
| `Combiner` | Converts t-out-of-N shares to t-out-of-t additive shares via Lagrange interpolation |

The trick: each party evaluates F(a, S(alpha_i) * l_i) locally (not F(a, S(alpha_i)) then multiply by l_i), avoiding large error amplification from Lagrange coefficients.

#### Scheme-Specific Multiparty (`multiparty/mpbgv/`, `multiparty/mpckks/`)

| Package | Features |
|---------|----------|
| `mpbgv` | BGV threshold refresh, sharing, transform |
| `mpckks` | CKKS threshold refresh, sharing, transform |

Both provide interactive bootstrapping for their respective schemes.

#### Protocol Infrastructure

- **CRS** (Common Reference String): Interface wrapping `sampling.PRNG` -- all parties share the same seed
- **CRP** (Common Random Polynomial): Sampled from CRS for each protocol round
- **Noise Flooding**: Statistical security parameter for key-switching/decryption (prevents leakage through error)
- **Serialization**: All shares implement `io.WriterTo`, `io.ReaderFrom`, `encoding.BinaryMarshaler/Unmarshaler`

## MHE-MPC Protocol Flow

The full Multiparty Homomorphic Encryption - Secure Multiparty Computation protocol:

```
1. Setup Phase (input-independent, reusable)
   1.1 Each party: keygen.GenSecretKeyNew() -> sk_i
   1.2 [if t<N] Thresholdizer: Shamir re-sharing of sk_i
   1.3 PublicKeyGenProtocol: CRS -> CRP -> GenShare -> Aggregate -> GenPublicKey
   1.4 RelinKeyGenProtocol: CRS -> CRP -> Round1 -> Aggregate -> Round2 -> Aggregate -> GenRelinKey
   1.5 GaloisKeyGenProtocol: CRS -> CRP -> GenShare -> Aggregate -> GenGaloisKey (per rotation)
   1.6 EvalKeyGenProtocol: CRS -> CRP -> GenShare -> Aggregate -> GenEvaluationKey

2. Evaluation Phase
   2.1 Input: parties encrypt with collective PK using standard Encoder + Encryptor
   2.2 Compute: standard single-party Evaluator on ciphertexts (can be outsourced)
   2.3 Output:
       a. KeySwitchProtocol or PublicKeySwitchProtocol: re-encrypt to receiver's key
       b. Receiver decrypts locally with standard Decryptor
```

## Key Generation API

```go
import (
    "github.com/luxfi/lattice/v7/core/rlwe"
    "github.com/luxfi/lattice/v7/schemes/ckks"
)

// Parameters from literal
params, _ := ckks.NewParametersFromLiteral(ckks.ExampleParameters128BitLogN14LogQP438)

// Key generation
keygen := rlwe.NewKeyGenerator(params)
sk := keygen.GenSecretKeyNew()
pk := keygen.GenPublicKeyNew(sk)
rlk := keygen.GenRelinearizationKeyNew(sk)
gk := keygen.GenGaloisKeyNew(galoisElement, sk)
evk := keygen.GenEvaluationKeyNew(skIn, skOut)

// Key pair shorthand
sk, pk := keygen.GenKeyPairNew()

// Custom Hamming weight secret
sk = keygen.GenSecretKeyWithHammingWeightNew(192)
```

## CKKS Usage Example

```go
import (
    "github.com/luxfi/lattice/v7/schemes/ckks"
    "github.com/luxfi/lattice/v7/core/rlwe"
)

params, _ := ckks.NewParametersFromLiteral(ckks.ExampleParameters128BitLogN14LogQP438)

keygen := rlwe.NewKeyGenerator(params)
sk, pk := keygen.GenKeyPairNew()
rlk := keygen.GenRelinearizationKeyNew(sk)

encoder := ckks.NewEncoder(params)
encryptor := rlwe.NewEncryptor(params, pk)
decryptor := rlwe.NewDecryptor(params, sk)
evaluator := ckks.NewEvaluator(params, rlwe.NewMemEvaluationKeySet(rlk))

// Encode and encrypt
values := []complex128{1.0+0i, 2.0+0i, 3.0+0i}
pt := encoder.EncodeNew(values, params.MaxLevel())
ct := encryptor.EncryptNew(pt)

// Homomorphic addition
ctResult := evaluator.AddNew(ct, ct)

// Decrypt and decode
ptResult := decryptor.DecryptNew(ctResult)
result := encoder.Decode(ptResult, params.LogSlots())
```

## DAG Scheduler Usage

```go
import "github.com/luxfi/lattice/v7/dag"

b := dag.NewBuilder()
b.Input(5)           // Node 1: load ct at level 5
id1 := b.LastNodeID()
b.Input(5)           // Node 2: load ct at level 5
id2 := b.LastNodeID()
b.Add(id1, id2)      // Node 3: add
id3 := b.LastNodeID()
b.Rescale(id3)        // Node 4: rescale
b.Output()

graph, _ := b.Build()
scheduler := dag.NewScheduler(graph, dag.WithFuseSimilarOps(true))
schedule, _ := scheduler.Schedule()
stats := schedule.Stats()
// stats.CriticalPath, stats.MaxBatchSize, stats.Parallelism
```
